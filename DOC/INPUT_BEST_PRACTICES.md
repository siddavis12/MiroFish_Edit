# MiroFish 입력 데이터 베스트 프랙티스

> 시드 문서의 품질이 시뮬레이션 전체 품질을 결정합니다. "Garbage In, Garbage Out"

---

## 1. 지원 형식 & 기본 제약

| 항목 | 값 |
|------|-----|
| 지원 포맷 | PDF, Markdown (.md), Plain Text (.txt) |
| 최대 업로드 | 50MB / 요청 |
| 복수 파일 | 지원 (파일별로 구분 마커 삽입 후 연결) |
| 청킹 | 500자 단위, 50자 오버랩, 문장 경계 스마트 분할 |
| 인코딩 | UTF-8 권장 (자동 감지 폴백 있음) |

## 2. 좋은 시드 문서의 5가지 조건

### ① 구체적인 이름이 있는 이해관계자 (Named Stakeholders)

```
❌ 나쁜 예: "한 기술 기업의 경영진이 우려를 표명했다"
✅ 좋은 예: "삼성전자 이재용 회장이 반도체 수출 규제에 대해 우려를 표명했다"
```

**이유**: MiroFish는 엔티티를 추출하여 시뮬레이션 에이전트로 만듭니다. "한 기업"은 에이전트가 될 수 없지만 "삼성전자"는 고유한 페르소나를 가진 에이전트가 됩니다.

### ② 관계와 상호작용이 명시된 텍스트 (Explicit Relationships)

```
❌ 나쁜 예: "여러 이해관계자가 관련되어 있다"
✅ 좋은 예: "환경부가 규제안을 발표하자, 현대자동차는 반발 성명을 냈고,
   그린피스 한국지부는 지지 입장을 표명했다"
```

**이유**: `OPPOSES`, `SUPPORTS`, `REGULATES` 같은 관계 엣지가 추출되어 시뮬레이션에서 에이전트 간 상호작용 패턴을 결정합니다.

### ③ 다양한 유형의 행위자 (Diverse Actor Types)

시스템은 정확히 **10개 엔티티 타입**을 온톨로지로 생성합니다. 다양한 유형이 문서에 등장해야 풍부한 온톨로지가 만들어집니다.

| 행위자 유형 | 시뮬레이션 특성 | 예시 |
|-------------|---------------|------|
| 정부/공공기관 | 낮은 활동량, 높은 영향력, 업무시간 활동 | 환경부, 교육부 |
| 미디어/언론 | 중간 활동량, 높은 영향력, 종일 활동 | 조선일보, KBS |
| 전문가/교수 | 중간 활동량, 중간 영향력 | 서울대 김OO 교수 |
| 기업/기관 | 낮은 활동량, 높은 영향력, 업무시간 활동 | 삼성전자, 현대차 |
| 시민/학생 | 높은 활동량, 낮은 영향력, 저녁 활동 | 대학생 커뮤니티 |
| NGO/시민단체 | 낮은 활동량, 높은 영향력 | 그린피스, 참여연대 |

### ④ 대립하는 관점이 포함된 내용 (Opposing Viewpoints)

```
✅ 이상적인 문서 구성:
- 문서 A: 정부 규제 찬성 입장 + 근거
- 문서 B: 기업 측 반대 입장 + 근거
- 문서 C: 시민사회 반응 + 다양한 의견
```

**이유**: 에이전트의 `stance` (supportive/opposing/neutral/observer)와 `sentiment_bias`가 문서에서 추출된 관점에 기반합니다. 단일 시각만 있으면 모든 에이전트가 같은 의견을 가져 시뮬레이션이 무의미해집니다.

### ⑤ 사건의 시간 흐름과 인과관계 (Temporal & Causal Context)

```
❌ 나쁜 예: "반도체 규제가 논란이다"
✅ 좋은 예: "2025년 3월 미국이 대중국 반도체 수출 규제를 강화한 이후,
   SK하이닉스의 주가가 15% 하락했고, 이에 산업통상자원부가
   대응 방안 마련에 나섰다"
```

**이유**: 시간 흐름과 인과 정보가 에이전트의 "개인 기억(personal memory)"으로 변환되어 시뮬레이션 중 더 맥락적인 발언을 하게 됩니다.

## 3. 문서 준비 실전 가이드

### 권장 분량

| 항목 | 권장값 | 이유 |
|------|--------|------|
| 문서 수 | 3~10개 | 다양한 관점 확보 + 청킹 품질 |
| 문서당 길이 | 500~3,000단어 | LLM 추출 컨텍스트 최적 범위 |
| 총 텍스트량 | ~50,000자 이내 | 온톨로지 생성 시 50KB 컷오프 |
| 목표 엔티티 수 | 20~100개 | 시뮬레이션 현실감 + 토큰 효율 |

### 문서 소싱 전략

#### 뉴스 기사 기반 (가장 효과적)

1. **핵심 사건 기사** 1~2편: 사건 개요, 주요 행위자, 타임라인
2. **각 이해관계자 반응 기사** 2~4편: 정부, 기업, 시민단체, 전문가 반응
3. **분석/칼럼** 1~2편: 배경 맥락, 향후 전망, 대립 구도 정리

#### 보고서/백서 기반

1. **공식 보고서** 1편: 정책 개요, 관계 기관 목록
2. **반대 의견서** 1편: 비판적 관점
3. **미디어 보도** 2~3편: 주요 반응과 여론

#### 학술/연구 기반

1. **연구 논문** 1~2편: 주제 배경, 전문가 관점
2. **뉴스 보도** 2~3편: 사회적 반응, 다양한 이해관계자

### 문서 전처리 체크리스트

```
□ 인코딩이 UTF-8인가?
□ 표/이미지가 제거되고 텍스트만 남았는가? (PDF의 경우)
□ 모든 주요 행위자가 실명(또는 구체적 명칭)으로 등장하는가?
□ 최소 3가지 다른 유형의 행위자가 포함되어 있는가?
□ 행위자 간 관계/상호작용이 명시되어 있는가?
□ 서로 다른 입장/관점이 포함되어 있는가?
□ 사건의 시간 흐름이 기술되어 있는가?
□ 추상적 개념보다 구체적 행위자 중심인가?
```

## 4. 시드 문서 수집을 위한 LLM 프롬프트 가이드

MiroFish에 넣을 좋은 시드 데이터를 찾거나 정제하기 위해 LLM을 활용하는 방법입니다.

### 4-1. 주제에서 시드 문서용 리서치 쿼리 생성

```
당신은 소셜 미디어 시뮬레이션을 위한 시드 문서를 수집하는 리서치 전문가입니다.

주제: [여기에 시뮬레이션하고 싶은 주제]

이 주제에 대해 다음을 포함하는 검색 쿼리 목록을 만들어주세요:

1. 핵심 사건 개요 검색 쿼리 (2개)
   - 사건의 타임라인과 주요 행위자가 드러나는 기사를 찾기 위한 쿼리

2. 이해관계자별 반응 검색 쿼리 (4~6개)
   - 정부/규제기관 입장
   - 직접 관련 기업/기관 입장
   - 시민단체/NGO 반응
   - 전문가/학계 의견
   - 일반 시민/소비자 반응
   - 해외 반응 (해당 시)

3. 분석/배경 검색 쿼리 (2개)
   - 대립 구도와 향후 전망이 정리된 분석 기사

각 쿼리에 대해 왜 이 쿼리가 시뮬레이션 품질에 도움이 되는지 한 줄로 설명해주세요.
```

### 4-2. 원본 문서를 MiroFish 최적 형태로 정제

```
당신은 소셜 미디어 시뮬레이션 엔진의 입력 데이터를 최적화하는 전문가입니다.

다음 문서를 아래 기준에 맞게 정제해주세요:

## 정제 기준
1. **행위자 명확화**: 모호한 표현("관계자", "한 기업")을 가능한 구체적 이름으로 변환
2. **관계 명시화**: 행위자 간 관계를 명확히 서술 (누가 누구에게 무엇을 했는지)
3. **입장/관점 보존**: 각 행위자의 찬반 입장과 근거를 명확히 유지
4. **시간 흐름 정리**: 사건의 순서를 시간순으로 재구성
5. **추상 개념 제거**: "여론", "분위기", "트렌드" 같은 추상적 개념을
   구체적 행위자의 행동/발언으로 대체
6. **불필요한 요소 제거**: 광고, 관련 기사 링크, 기자 정보 등 제거

## 출력 형식
- 마크다운 또는 플레인 텍스트
- 500~3,000단어
- UTF-8 인코딩

## 원본 문서:
[여기에 원본 텍스트 붙여넣기]
```

### 4-3. 주제에 대한 시드 문서 직접 생성 (데이터가 부족할 때)

```
당신은 소셜 미디어 여론 시뮬레이션을 위한 시드 문서를 작성하는 전문가입니다.

## 시뮬레이션 주제
[주제 설명]

## 작성 요구사항

다음 구조의 시드 문서 세트를 작성해주세요:

### 문서 1: 사건 개요 (800~1,200단어)
- 사건의 배경, 경과, 현재 상황
- 최소 5명의 구체적 행위자 (실명 또는 가상의 구체적 이름)
- 각 행위자의 역할과 입장 명시
- 시간순 사건 전개

### 문서 2: 찬성 측 관점 (500~800단어)
- 찬성 측 주요 행위자 3명 이상
- 각자의 논거와 근거
- 행위자 간 연대/협력 관계

### 문서 3: 반대 측 관점 (500~800단어)
- 반대 측 주요 행위자 3명 이상
- 각자의 반대 논거와 근거
- 행위자 간 연대/협력 관계

### 문서 4: 중립/관찰자 관점 (400~600단어)
- 언론, 전문가, 일반 시민 등
- 양측을 분석하는 관점
- 향후 전망

## 중요 규칙
- 모든 행위자는 소셜 미디어에서 활동할 수 있는 실체여야 함
  (사람, 기관, 기업 등 — "여론", "트렌드" 같은 추상 개념 불가)
- 행위자 간 관계를 명시적으로 서술할 것
  (예: "A는 B의 규제에 반대하며 C와 공동 성명을 발표했다")
- 각 행위자의 소셜 미디어 활동 특성을 암시할 것
  (예: "Z교수는 자신의 블로그에서 상세한 분석을 자주 올리는 것으로 알려져 있다")
```

### 4-4. 시뮬레이션 요구사항 설명 작성 (Step1에서 필요)

MiroFish는 온톨로지 생성 시 "시뮬레이션 요구사항 설명"을 입력받습니다. 이 설명의 품질이 온톨로지 품질을 직접적으로 좌우합니다.

```
시뮬레이션 요구사항 설명을 작성해주세요. 다음 항목을 포함해야 합니다:

주제: [시뮬레이션 주제]

1. **시뮬레이션 목적**: 무엇을 관찰/예측하고 싶은가?
2. **핵심 질문**: 시뮬레이션으로 답하고 싶은 구체적 질문 2~3개
3. **주요 행위자 그룹**: 시뮬레이션에 참여해야 할 행위자 유형
4. **관찰 포인트**: 어떤 상호작용 패턴을 관찰하고 싶은가?
5. **기대 시나리오**: 예상되는 2~3가지 전개 방향

형식: 자연어 문단, 200~500단어
언어: 영어 (시스템 프롬프트가 영어이므로)
```

## 5. 시뮬레이션 요구사항 작성 팁

온톨로지 생성 프롬프트는 **영어**로 동작합니다. 시뮬레이션 요구사항도 영어로 작성하면 더 좋은 결과를 얻습니다.

### 좋은 시뮬레이션 요구사항 예시

```
Simulate the public discourse on South Korea's semiconductor export
regulation tightening (March 2025).

Key questions:
- How do different stakeholder groups (government, industry, academia,
  civil society) frame the issue on social media?
- Does the discourse polarize over time, or does consensus emerge?
- Which actors become opinion leaders in the discussion?

Expected actor types: Government officials (Ministry of Trade),
semiconductor companies (Samsung, SK Hynix), industry associations
(KSIA), university researchers, financial analysts, tech journalists,
and general public commentators.

Observe: Formation of pro/anti-regulation camps, information cascading
patterns, and whether expert opinions influence public sentiment
trajectories.
```

### 나쁜 시뮬레이션 요구사항 예시

```
반도체에 대한 여론을 시뮬레이션해주세요.
```

→ 너무 모호하여 온톨로지가 일반적인 엔티티만 생성하게 됩니다.

## 6. 흔한 실수와 해결책

| 실수 | 증상 | 해결책 |
|------|------|--------|
| 추상적 개념 위주 문서 | 엔티티가 거의 추출 안됨 | 구체적 행위자 중심으로 문서 재작성 |
| 단일 관점 문서만 입력 | 모든 에이전트가 같은 입장 | 대립 관점 문서 추가 |
| 너무 긴 단일 문서 | 50KB 컷오프로 뒷부분 무시됨 | 3~10개 짧은 문서로 분할 |
| 표/차트가 많은 PDF | 텍스트 추출 품질 저하 | 텍스트 위주 문서로 변환 |
| 행위자 이름 불일치 | 같은 사람이 다른 엔티티로 중복 | 문서 내 이름 통일 (약칭 정리) |
| 영문+한글 혼용 이름 | "Samsung"과 "삼성" 별개 추출 | 하나의 표기로 통일 |
| 관계 암시만 있음 | 그래프 엣지가 부족 | 관계를 명시적 문장으로 서술 |

## 7. 입력 데이터 품질 자가 진단

시드 문서를 준비한 후, 아래 점수표로 품질을 자가 진단하세요:

| 항목 | 배점 | 기준 |
|------|------|------|
| 구체적 행위자 수 | /20 | 10명 미만=5, 10~20명=10, 20~50명=15, 50명+=20 |
| 행위자 유형 다양성 | /20 | 2유형=5, 3~4유형=10, 5~6유형=15, 7유형+=20 |
| 명시적 관계 수 | /20 | 5개 미만=5, 5~15개=10, 15~30개=15, 30개+=20 |
| 관점 다양성 | /15 | 단일=0, 2개=5, 3개=10, 4개+=15 |
| 시간 흐름 정보 | /10 | 없음=0, 부분=5, 상세=10 |
| 문서 수/분량 적절성 | /10 | 1편=3, 2편=5, 3~5편=8, 5~10편=10 |
| 이름 일관성 | /5 | 불일치 있음=0, 완전 일관=5 |

- **80점 이상**: 우수 — 풍부한 시뮬레이션 기대
- **60~79점**: 양호 — 기본적인 시뮬레이션 가능
- **40~59점**: 보통 — 일부 보완 필요
- **40점 미만**: 미흡 — 문서 보강 권장

## 8. 주제별 시드 문서 템플릿

### 정책/규제 이슈

```
필요 문서:
1. 정책 발표 보도 (규제 기관, 내용, 배경)
2. 규제 대상 기업/산업 반응
3. 시민단체/전문가 의견
4. 해외 사례/반응 (비교 관점)
5. 종합 분석 기사
```

### 기업/산업 이슈

```
필요 문서:
1. 사건 발단 보도 (기업, 사건 개요)
2. 해당 기업 공식 입장
3. 경쟁사/업계 반응
4. 소비자/투자자 반응
5. 전문가/애널리스트 분석
```

### 사회/문화 이슈

```
필요 문서:
1. 이슈 발생 보도
2. 찬성 측 주요 인물/단체 입장
3. 반대 측 주요 인물/단체 입장
4. 언론/미디어 논조 분석
5. 전문가(사회학자, 법학자 등) 의견
```
